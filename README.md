# Auto.ru parser

Парсер объявлений с Auto.ru, доработанный под сбор не только ссылок, но и расширенных характеристик автомобилей.

## Что умеет

Скрипт работает в два этапа:

1. Собирает ссылки на объявления из выдачи Auto.ru
2. Переходит в каждое объявление и сохраняет детальные данные в CSV

## Какие данные собираются

По каждому объявлению могут собираться:

- ссылка на объявление
- марка
- регион
- title объявления
- цена
- данные блока «Владение»
- технические характеристики
- параметры двигателя
- батарея и запас хода
- размеры, масса, подвеска, трансмиссия
- эксплуатационные показатели

Итог сохраняется в CSV-файл.

## Структура проекта

- `main.py` — базовый входной файл
- `src/main_pipeline.py` — полный пайплайн: сбор ссылок + сбор деталей
- `src/parser.py` — сбор ссылок на объявления
- `src/collect_details.py` — сбор детальных характеристик по ссылкам
- `data/links.txt` — промежуточный список ссылок
- `data/electro_cars_details.csv` — итоговый файл с деталями

## Особенности

- используется Selenium, так как Auto.ru активно защищается от ботов
- часть защиты может требовать ручного прохождения капчи
- для стабильной работы используется пользовательский Chrome-профиль
- поддерживается промежуточное сохранение результатов
- при повторном запуске можно продолжать сбор с места остановки

## Установка

### 1. Установить `uv`

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
2. Установить зависимости
uv sync
3. Создать .env

Создай .env на основе .env_example и заполни нужные значения.

Особенно важно:

HEADERS

COOKIE

USE_SELENIUM

Запуск
Полный запуск: ссылки + детали
uv run src/main_pipeline.py

После запуска нужно вставить ссылку на выдачу Auto.ru, например:

https://auto.ru/cars/all/engine-electro/
Отдельно только сбор деталей по уже готовым ссылкам

Если файл data/links.txt уже есть:

uv run src/collect_details.py

Результаты

Основной итоговый файл:

data/electro_cars_details.csv

Файл прогресса для продолжения после остановки:

data/electro_cars_details_progress.json
Важное замечание

Проект основан на существующей заготовке парсера и был существенно доработан под конкретную задачу:

сбор ссылок через Selenium

сбор детальных характеристик из карточек объявлений

промежуточное сохранение

продолжение после прерывания

добавление бренда, региона и расширенных полей

полный пайплайн в одном запуске
